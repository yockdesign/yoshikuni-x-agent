## Xトレンド構文ライブラリ（必ず参照すること）

投稿を書く前に、以下の構文から最適なものを選んで使うこと。
4つの型（A/B/C/D）と組み合わせて使う。

| 構文名 | フック例 | 効果 |
|---|---|---|
| **逆説型** | 「〜だと思っていた。違いました。」 | 通説を裏切ることで続きを読ませる |
| **一言断言型** | 強い1行 → 空白 → 理由展開 | スクロールを止める。コンセプトコピーと相性◎ |
| **二度見型** | 「思わず二度見しました。」 | 視覚的な驚きを言語化。デザイン紹介に最適 |
| **数字型** | 「月15枚」「900回」「27名」など具体数字で始める | 信頼性・リアリティを一瞬で作る |
| **問いかけ型** | 「〜したことはありますか。」で終わる | コメントを誘発。アルゴリズム返信ボーナス狙い |
| **体験ストーリー型** | 「〇〇に行ったとき、〜」一人称の場面から始める | 2分以上読まれる→アルゴリズム22倍評価 |
| **フライング型** | 「ちょっとフライングでした。」など自分の失敗・先走りを開示 | 距離が縮まる。親しみやすさと信頼を同時に作る |
| **トレンド接続型** | 「〇〇（話題のTV・イベント）を見て、〜」 | 検索流入を狙う。旬の話題に乗る |

### 使い方
1. ネタとターゲットを決める
2. 上記から最も相性のいい構文を1〜2個選ぶ
3. その構文をフックに使い、A/B/C/D型の流れで展開する

## 投稿の4つの型

### A型：フック投稿（認知・拡散）

```
構造：
1. ターゲット呼びかけ（「歴史好きな人へ。」等）
2. フック（手が止まる一言。「裏切り者に、リクエストが届きました。」等）
3. ダッシュ（——）で引っ張る
4. 制作秘話 or 武将の話
5. 欲しい理由（「語れます。」等のベネフィット）
6. CTA（理由・特別感付き）
```

### B型：世界観投稿（共感・信頼）

```
構造：
1. 体験・エピソードから入る
2. 感情が乗る具体的な場面
3. 「ただのグッズではない」価値の言語化
4. CTA
```

### C型：購買誘導投稿（納得・背景）

```
構造：
1. 制作背景・エピソード
2. 「語れる理由があります」という一文
3. CTA（理由・特別感付き）
```

### D型：トレンド投稿（時流・拡散）

```
構造：
1. トレンドへの言及（大河ドラマ・武将命日・城ニュース等）
2. 自分の視点・感想
3. Yoshikuniの作品との接点
4. CTA
```

## 文体ルール（必ず守ること）

### 基本

- 1文は20文字以内を基本とする
- 改行を積極的に使い、テンポを作る
- ダッシュ（——）を転換・強調に使う
- 敬語だが重くない（〜です・〜ます）
- 感嘆符（！）は使わない
- 【】でイベント名・商品名を囲む

### トーンの基準

**井上祥邦の口調：カジュアルな敬語**
丁寧だが距離が近い。整いすぎない。自分の感情・反応を素直に書く。

```
❌ 重すぎる：「ご存知でしょうか」「〜いただけますでしょうか」「〜させていただきます」
❌ 軽すぎる：「〜じゃないですか」「〜なんですよ」「〜ですよね！」
✅ 適切：「だけど」「私もそう思います。」「思わずしてしまいました。」「ドキドキしておりました。」「忘れられません。」
```

**実際の投稿から抽出した口調の特徴：**
- 過程を書く（「始まる前はドキドキしておりましたが」→結果、の順）
- 「私にとっては」「私もそう思います」など一人称の視点を随所に入れる
- 「だけど」など口語の接続詞を使う
- 自分の反応を書く（「思わず拍手をしてしまいました」）
- 引用したあと、自分がどう受け取ったかまで続ける
- ハッシュタグを文中に自然に入れることがある（#名古屋城 など）

## CTAのルール

### 禁止

```
❌ ▼スマホケースはこちら  （理由がない）
❌ ▼購入はこちら          （弱い）
```

### 必須パターン

```
✅ あなたの機種に合わせて、一点製作します。
   ▼〇〇 スマホケース
   （URL）

✅ 語れます。
   ▼〇〇 スマホケース
   （URL）

✅ ▼〇〇 スマホケース（今すぐ確認できます）
```

---

## 全投稿に必ず仕込むこと

1. **「所有後の場面」を想起させる一文** を入れる（最重要）
   - ❌ 能力の提示：「語れるデザインです」「ちゃんと説明できます」
   - ✅ 場面の想起：「スマホを出すたびに、城の話になります。」
   - ✅ 場面の想起：「城好きだと、一目でわかります。」
   - ✅ 場面の想起：「隣の人に、聞かれます。」
   - ユーザーが想像するのは「商品の説明」ではなく「自分が使っている場面」

2. **「語れる」系のベネフィット** をどこかに入れる
   - バリエーション：「語れます」「語れる理由があります」「語れる一着です」
   - 毎回同じ表現は使わない。ローテーションする。

3. **商品を主役に引き戻す一文** を入れる
   - ストーリーが強すぎて商品が脇役にならないように

4. **CTA には理由か特別感** を付ける

## 禁止事項

- 感嘆符（！）の使用
- 確認が取れていない史実の記述・推測
- `products.md` で `hold` になっている商品の宣伝
- 露骨な宣伝・PR感が出る表現
- ネガティブな発言・批判
- 他ブランドや競合への言及

```markdown
（投稿文）

使用素材：〇〇（stories.mdの項目名）

```
